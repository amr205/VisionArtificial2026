              ¿Por qué importa el tamaño de salida?

              En una red profunda, cada capa convolucional transforma la dimensión espacial del tensor. Calcular el tamaño correcto evita errores de dimensión al conectar capas (especialmente antes de capas Flatten o Linear). La fórmula general unifica todos los parámetros: un stride mayor reduce el tamaño, el padding lo conserva, y la dilatación expande el campo receptivo sin aumentar parámetros.

import torch.nn as nn

# Kernels comunes

# → más campo pero más parámetros

conv7x7 = nn.Conv2d(3, 64, kernel_size=7, stride=2)
# → típico en la primera capa (ResNet, etc.)

# Parámetros = k² × C_in × C_out (+ bias)
import torch
x = torch.randn(1, 64, 28, 28)
y3 = conv3x3(x)  # → (1, 64, 26, 26)
y5 = conv5x5(x)  # → (1, 64, 24, 24)

              Regla práctica:

              En la mayoría de redes modernas (ResNet, EfficientNet, MobileNet) prácticamente todos los kernels son 3×3. Los kernels 1×1 se usan para mezclar canales sin coste espacial (bottleneck). Los kernels grandes (7×7) solo aparecen en la primera capa para reducir rápidamente la resolución de la imagen de entrada.

// Python · padding

            
# Sin padding: la salida se reduce
conv_valid = nn.Conv2d(1, 1, kernel_size=3, padding=0)
# 5×5 entrada → 3×3 salida

# Con padding=1: conserva tamaño (k=3)
conv_same = nn.Conv2d(1, 1, kernel_size=3, padding=1)
# 5×5 entrada → 5×5 salida

# PyTorch: padding='same' (stride=1 obligatorio)
conv_auto = nn.Conv2d(1, 1, kernel_size=3,
                        padding='same')

# Tipos de padding en PyTorch
# padding_mode='zeros'     (default)
# padding_mode='reflect'
# padding_mode='replicate'
# padding_mode='circular'

# Regla: padding = (k-1)//2 conserva tamaño
def same_padding(k): return (k - 1) // 2

              ¿Cuándo usar padding?

              En la mayoría de capas intermedias se usa padding="same" para preservar la resolución y permitir que las capas de pooling o strided convolutions sean los únicos responsables de la reducción espacial. Sin padding, la imagen se encoge con cada capa y los píxeles del borde tienen menos influencia en la salida.

# stride=1 (default) — sin reducción

                      stride=1, padding=1)

conv_s2 = nn.Conv2d(64, 128, kernel_size=3,
                      stride=2, padding=1)

# Equivalente con MaxPool
maxpool_equiv = nn.Sequential(
    nn.Conv2d(64, 128, kernel_size=3, padding=1),
    nn.MaxPool2d(kernel_size=2, stride=2)
)
# Ambos: 28×28 → 14×14

# Primera capa de ResNet
first_layer = nn.Conv2d(3, 64, kernel_size=7,
                           stride=2, padding=3)
# 224×224 → 112×112

              Strided Conv vs MaxPool:

              Redes modernas como ResNet y EfficientNet usan convoluciones con stride=2 en lugar de MaxPooling para reducir la resolución, porque la convolución tiene parámetros que se aprenden durante el entrenamiento mientras que MaxPool es una operación fija. Esto da más flexibilidad al modelo para aprender cómo submuestrear.

// Python · dilation

            
# Convolución dilatada (atrous)
import torch.nn as nn

# d=1: convolución normal
conv_d1 = nn.Conv2d(64, 64, kernel_size=3,
                      dilation=1, padding=1)

# d=2: campo receptivo 5×5 con 9 params
conv_d2 = nn.Conv2d(64, 64, kernel_size=3,
                      dilation=2, padding=2)
# padding = dilation para conservar tamaño

# d=4: campo receptivo 9×9
conv_d4 = nn.Conv2d(64, 64, kernel_size=3,
                      dilation=4, padding=4)

# ASPP (Atrous Spatial Pyramid Pooling)
# usado en DeepLab v3 para segmentación
aspp = nn.ModuleList([
    nn.Conv2d(256, 256, 3, dilation=6,  padding=6),
    nn.Conv2d(256, 256, 3, dilation=12, padding=12),
    nn.Conv2d(256, 256, 3, dilation=18, padding=18),
])

              ¿Por qué usar dilatación?

              La dilatación permite expandir el campo receptivo exponencialmente sin aumentar el número de parámetros ni reducir la resolución. Con d=1,2,4,8 apiladas, el campo receptivo crece como 3, 7, 15, 31 píxeles. Esto es especialmente valioso en segmentación semántica donde se necesita contexto amplio pero se quiere mantener la resolución espacial alta para localizar con precisión.

// Python · in_channels y out_channels

            
# Imagen RGB: 3 canales de entrada
conv1 = nn.Conv2d(
    in_channels=3,   # R, G, B
    out_channels=64,  # 64 filtros aprendibles
    kernel_size=3,
    padding=1
)
# x: (N, 3, H, W)  →  y: (N, 64, H, W)
# weight.shape = (64, 3, 3, 3)
# bias.shape   = (64,)
# total params = 64×3×3×3 + 64 = 1792

# Bloque típico: duplicar canales al bajar resolución
conv2 = nn.Conv2d(64, 128, 3, stride=2, padding=1)
conv3 = nn.Conv2d(128, 256, 3, stride=2, padding=1)

# Contar parámetros manualmente
def conv_params(Cin, Cout, k, bias=True):
    w = Cout * Cin * k * k
    b = Cout if bias else 0
    return w + b

print(conv_params(3, 64, 3))  # → 1792
print(conv_params(64, 128, 3)) # → 73856

              Intuición de los canales:

              La entrada a una convolución es siempre un volumen 3D (Cin × H × W), no una imagen 2D. Cada filtro es también un volumen (Cin × k × k) que se desliza espacialmente y produce un solo mapa 2D. Si quieres Cout mapas distintos, necesitas Cout filtros distintos. Por eso el tensor de pesos tiene 4 dimensiones: (Cout, Cin, k, k). La primera capa de una red toma imágenes RGB con Cin=3; las capas intermedias pueden tener Cin=64, 128, 256, etc.
