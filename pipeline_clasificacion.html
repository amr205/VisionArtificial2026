// ¿Qué es un pipeline de clasificación?

Un pipeline de clasificación de imágenes es la secuencia completa de pasos para tomar imágenes crudas y producir un modelo capaz de categorizarlas. Cada etapa transforma o prepara los datos y el modelo para la siguiente. Descuidar cualquier paso produce modelos que no generalizan.

La práctica moderna consiste en partir de un modelo preentrenado en ImageNet (ResNet, EfficientNet, MobileNet…) y adaptarlo mediante transfer learning + fine-tuning en dos fases: primero entrenar solo la nueva cabeza, luego descongelar las capas profundas con un learning rate muy bajo.

// Diagrama del flujo

Dataset splits
Train / Val / Test — nunca evalúes sobre datos de entrenamiento
Transfer Learning
Pesos preentrenados en ImageNet como punto de partida eficiente
Fase 1 — Head
Backbone congelado, solo entrena la nueva cabeza con LR ~1e-3
Fase 2 — Deep FT
Descongela últimas capas, LR 10–100× menor para no destruir pesos
Augmentation
Solo en train — regularizador implícito, mejora generalización
Early Stopping
Guarda el mejor checkpoint según val_accuracy, no el último epoch
Regla de oro: Empieza siempre con el backbone completamente congelado. Cuando la cabeza haya convergido, descongela progresivamente las capas más profundas con LR ~10× menor. Esto garantiza que los pesos de ImageNet no se destruyan al inicio del entrenamiento.

La convención más común es una carpeta por clase. PyTorch la detecta automáticamente con ImageFolder; Keras con image_dataset_from_directory. Divide siempre en train / val / test antes de cualquier preprocesamiento.

Lee la estructura de carpetas y asigna etiquetas automáticamente

// ¿Por qué normalizar con las stats de ImageNet?

Los modelos preentrenados en ImageNet esperan imágenes normalizadas con las medias y desviaciones de ImageNet. Si normalizas diferente, la distribución de activaciones en el backbone será completamente distinta a lo que los pesos esperan, y el transfer learning fallará.

El flujo es siempre: píxeles [0,255] → ToTensor convierte a [0,1] con forma (C,H,W) → Normalize centra y escala por canal.
mean ImageNet
[0.485, 0.456, 0.406] — por canal R, G, B
std ImageNet
[0.229, 0.224, 0.225] — por canal R, G, B
Fórmula
x_norm = (x − mean) / std, aplicada canal a canal
Eval estándar
Resize(256) → CenterCrop(224) — igual que evaluación original de ImageNet
Regla de oro: La normalización de val y test nunca incluye augmentation. Solo Resize + CenterCrop + ToTensor + Normalize. Las estadísticas son siempre las de ImageNet si usas transfer learning.
PyTorchtransforms.Normalize — stats ImageNet
from torchvision import transforms
import torch

MEAN = [0.485, 0.456, 0.406]
STD  = [0.229, 0.224, 0.225]

# Transform SOLO para VAL / TEST
# (sin ninguna augmentación)
val_transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),       # → [0,1], (C,H,W)
    transforms.Normalize(
        mean=MEAN, std=STD
    ),
])

# Aplicar al dataset de val/test
val_ds.transform  = val_transform
test_ds.transform = val_transform

# Utilidad: deshacer normalización (visualizar)
def denormalize(tensor):
    # tensor: (C, H, W) ya en CPU
    m = torch.tensor(MEAN).view(3,1,1)
    s = torch.tensor(STD).view(3,1,1)
    return (tensor * s + m).clamp(0, 1)

// ¿Qué es y por qué funciona?

La aumentación de datos aplica transformaciones aleatorias durante el entrenamiento: el modelo ve versiones distintas de la misma imagen en cada epoch. Actúa como regularizador implícito forzando al modelo a aprender características invariantes a rotaciones, flips, cambios de color, etc.

Usa aumentaciones realistas para tu dominio. Si clasificas células microscópicas, las rotaciones 360° tienen sentido. Si clasificas textos escritos, el flip vertical no lo tiene.
RandomHorizontalFlip
Flip horizontal p=0.5 — casi siempre útil en imágenes naturales
RandomResizedCrop
Crop + resize aleatorio — el modelo no ve siempre el centro completo
ColorJitter
Varía brillo, contraste, saturación y tono — bueno para variación de iluminación
RandomRotation
Rotaciones pequeñas ±15° para robustez a la orientación
RandAugment
Política aprendida automáticamente — torchvision lo incluye listo
MixUp / CutMix
Técnicas avanzadas que mezclan imágenes — muy efectivas para datasets pequeños
⚠ Solo en TRAIN: Val y test siempre usan transformaciones fijas (Resize + CenterCrop + Normalize) para que la evaluación sea reproducible y comparable entre experimentos.
PyTorchtransforms — pipeline TRAIN completo
from torchvision import transforms, datasets

MEAN = [0.485, 0.456, 0.406]
STD  = [0.229, 0.224, 0.225]

train_transform = transforms.Compose([
    transforms.RandomResizedCrop(224),

    transforms.ToTensor(),

val_transform = transforms.Compose([

    transforms.CenterCrop(224),

# Aplicar transforms distintos

    'dataset/train', transform=train_transform)

    'dataset/val',   transform=val_transform)

train_aug_v2 = transforms.Compose([

def augment(image, label):

// Transfer Learning: reutilizar el conocimiento

El transfer learning toma una red entrenada en ImageNet (1.2M imágenes, 1000 clases) y la adapta a tu tarea. El backbone ha aprendido detectores de bordes, texturas y formas útiles para casi cualquier problema de visión.

El proceso: (1) Cargar el modelo sin la cabeza final (include_top=False en Keras); (2) Congelar todos los pesos del backbone; (3) Añadir una nueva cabeza con tu número de clases + opcional Dropout.
Backbone / Base
Todo menos la última capa FC — extractor de features jerárquicos
Head / Cabeza
Nuevas capas FC que producen la predicción para TUS clases
requires_grad=False
Congela parámetros en PyTorch — no participan en backprop
trainable=False
Equivalente en Keras — congela todas las capas del backbone
GlobalAveragePooling2D
Colapsa el mapa espacial a un vector — sustituye al Flatten en redes modernas
Dropout
Regularización en la cabeza — p=0.2 a 0.5 según el tamaño del dataset
¿Qué modelo elegir? ResNet50 = clásico confiable. EfficientNetV2-S = mejor precisión/eficiencia. MobileNetV3 = cuando necesitas velocidad. ViT-B/16 = SOTA con muchos datos. Para <1k imágenes, congela más capas y usa más Dropout.
PyTorchResNet50 — reemplazar fc + congelar backbone
import torch
import torch.nn as nn
from torchvision import models

NUM_CLASSES = 10
device = ('cuda' if torch.cuda.is_available()
          else 'cpu')

# Cargar ResNet50 preentrenado
model = models.resnet50(
    weights=models.ResNet50_Weights.DEFAULT
)

# ── FASE 1: congelar backbone ──────
for param in model.parameters():
    param.requires_grad = False

# Reemplazar la capa de clasificación
# ResNet50.fc = Linear(2048 → 1000)
in_f = model.fc.in_features   # 2048
model.fc = nn.Sequential(
    nn.Dropout(0.3),
    nn.Linear(in_f, NUM_CLASSES)
)
# Ahora solo model.fc tiene grad=True
model = model.to(device)

# Otros modelos — propiedad equivalente:
# EfficientNet: model.classifier[1]
# MobileNetV3:  model.classifier[3]
# DenseNet:     model.classifier
# ViT:          model.heads.head

# Verificar parámetros entrenables
total = sum(
    p.numel() for p in model.parameters())
train_p = sum(
    p.numel() for p in model.parameters()
    if p.requires_grad)
print(f"Total:      {total:>12,}")
print(f"Entrenables:{train_p:>12,}")
# Total: ~25M | Entrenables: ~20K

// Dos fases para no destruir los pesos preentrenados

Fase 1 — Feature extraction (5–15 epochs): Backbone congelado. Solo la cabeza entrena con LR normal (~1e-3). Rápido y seguro. El backbone actúa como extractor de features fijo.

Fase 2 — Fine-tuning profundo (10–30 epochs): Descongela las últimas N capas del backbone y sigue entrenando con LR muy bajo (~1e-5 a 1e-4). Adapta los features del backbone a tu dominio sin destruir lo aprendido. Usa LRs diferenciales: más bajo en capas antiguas, más alto en las nuevas.
LR Fase 1
~1e-3 — solo la cabeza, puede ser relativamente alto
LR Fase 2
~1e-5 a 1e-4 — backbone, extremadamente conservador
CosineAnnealingLR
Decae el LR suavemente en coseno — mejor que StepLR en fine-tuning
ReduceLROnPlateau
Reduce LR cuando val_loss no mejora en N epochs — adaptativo
Early Stopping
Detener cuando val_loss no mejora en patience epochs
Gradient Clipping
clip_grad_norm_(1.0) — estabiliza el entrenamiento en fase 2
¿Cuántas capas descongelar? Empieza con las últimas 2-3 capas (layer4 en ResNet, top_conv + top_bn en EfficientNet). Con más datos puedes descongelar más. Con <1k imágenes quédate en fase 1.
PyTorchBucle completo — 2 fases + early stopping
import torch.optim as optim

criterion = nn.CrossEntropyLoss()

# ── FASE 1 ────────────────────────────
optimizer = optim.AdamW(
    model.fc.parameters(),
    lr=1e-3, weight_decay=1e-4
)
scheduler = optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=10
)

def run_epoch(model, loader, opt, crit, train):
    model.train() if train else model.eval()
    loss_sum, correct = 0, 0
    ctx = torch.enable_grad() if train else torch.no_grad()
    with ctx:
        for imgs, lbls in loader:
            imgs, lbls = imgs.to(device), lbls.to(device)
            if train: opt.zero_grad()
            out  = model(imgs)
            loss = crit(out, lbls)
            if train:
                loss.backward()
                nn.utils.clip_grad_norm_(
                    model.parameters(), 1.0)
                opt.step()
            loss_sum += loss.item()
            correct += (out.argmax(1) == lbls).sum().item()
    n = len(loader.dataset)
    return loss_sum / len(loader), correct / n

# Fase 1 — 10 epochs (solo cabeza)
for ep in range(10):
    tr_l, tr_a = run_epoch(
        model, train_dl, optimizer, criterion, True)
    va_l, va_a = run_epoch(
        model, val_dl, None, criterion, False)
    scheduler.step()
    print(f"Ep{ep+1} train={tr_a:.3f} val={va_a:.3f}")

# ── FASE 2 ────────────────────────────
for layer in [model.layer3, model.layer4]:
    for p in layer.parameters():
        p.requires_grad = True

optimizer2 = optim.AdamW([
    {'params': model.layer3.parameters(), 'lr':1e-5},
    {'params': model.layer4.parameters(), 'lr':5e-5},
    {'params': model.fc.parameters(),     'lr':1e-4},
], weight_decay=1e-4)

best_acc, wait, patience = 0, 0, 5
for ep in range(30):
    run_epoch(model, train_dl, optimizer2, criterion, True)
    _, va_a = run_epoch(
        model, val_dl, None, criterion, False)
    if va_a > best_acc:
        best_acc = va_a
        torch.save(model.state_dict(), 'best.pth')
        wait = 0
    else:
        wait += 1
    if wait >= patience:
        print("Early stop"); break

// Métricas y cuándo usarlas

La accuracy es intuitiva pero engañosa con datasets desbalanceados. Con 95% de imágenes de clase A, un modelo trivial tiene 95% de accuracy. Usa siempre F1-score por clase y la matriz de confusión para entender qué clases se confunden.

La evaluación final siempre sobre el conjunto de test (no val). El conjunto de val solo se usa durante el diseño para seleccionar hiperparámetros.
MétricaFórmulaCuándo usar
AccuracyTP+TN / TotalDataset perfectamente balanceado
PrecisionTP / (TP+FP)Falsos positivos son costosos (spam)
RecallTP / (TP+FN)Falsos negativos son costosos (medicina)
F1-Score2·P·R / (P+R)Balance P/R — datasets desbalanceados
Macro F1Media F1 por claseTodas las clases igual de importantes
Weighted F1F1 ponderado por soporteClases con diferente número de ejemplos
Confusion Matrix Classification Report sklearn.metrics torchmetrics TTA (Test-Time Augmentation) curvas train/val

from sklearn.metrics import (

    confusion_matrix,

import matplotlib.pyplot as plt

# Test-Time Augmentation (TTA)
# Promedia N predicciones con crops/flips
def tta_predict(model, img, n=5):
    aug = transforms.Compose([
        transforms.RandomCrop(224),
        transforms.RandomHorizontalFlip()
    ])
    preds = [torch.softmax(
        model(aug(img).unsqueeze(0).to(device)),
        dim=1) for _ in range(n)]
    return torch.stack(preds).mean(0)

import numpy as np
import matplotlib.pyplot as plt
from sklearn.metrics import (
    classification_report, confusion_matrix,
    ConfusionMatrixDisplay
)

# Evaluación rápida con Keras
test_loss, test_acc = model.evaluate(test_ds, verbose=0)
print(f"Test Loss:     {test_loss:.4f}")
print(f"Test Accuracy: {test_acc:.4f}")

# Predicciones para sklearn
all_preds, all_labels = [], []
for imgs, labels in test_ds:
    probs = model.predict(imgs, verbose=0)
    all_preds.extend(np.argmax(probs, axis=1))
    all_labels.extend(labels.numpy())

all_preds  = np.array(all_preds)
all_labels = np.array(all_labels)
class_names = train_ds.class_names

# Classification report
print(classification_report(
    all_labels, all_preds,
    target_names=class_names
))

# Matriz de confusión
cm = confusion_matrix(all_labels, all_preds)
disp = ConfusionMatrixDisplay(
    cm, display_labels=class_names
)
fig, ax = plt.subplots(figsize=(8,8))
disp.plot(ax=ax, cmap='Blues',
          xticks_rotation='vertical')
plt.tight_layout()
plt.savefig('confusion_matrix.png', dpi=150)
